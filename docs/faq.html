FAQ

Frequently asked questions

Quick answers for common ClipVault questions.

What is ClipVault?

ClipVault is a Windows clipboard manager focused on history, pinned items, reusable snippets, tray support, and a polished desktop experience.

Does ClipVault work offline?

Yes. ClipVault is designed around local clipboard workflows and stores its working data on the machine.

Does ClipVault store my clipboard data locally?

Yes. Clipboard data, settings, and logs are stored locally. Review the privacy page for more detail about what may be included in local storage and backups.

Does ClipVault support more than clipboard history?

Yes. In addition to history, ClipVault supports pinned items, reusable snippets, settings controls, and a built-in log viewer.

Can I disable clipboard monitoring?

Yes. Clipboard monitoring can be turned on or off in Settings.

How do updates work?

Installed builds can use the packaged update flow when a release feed is configured. Source builds may not behave the same way.

Does ClipVault replace a password manager?

No. ClipVault is a clipboard tool, not a password manager. Sensitive clipboard content should always be handled with care.

Where can I get help or report an issue?

Use the GitHub repository for releases, documentation, and issue tracking as the project grows.
